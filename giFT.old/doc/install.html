1. Getting it

The easiest way to get the latest version of giFT/OpenFT is through CVS. Actually, at the moment, there is no other way :)

Go to the directory you want to download the sources to (/usr/src or ~/cvs?) and do (the $ is used to indicate this is a shell command):

  $ cvs -z3 -d:pserver:anonymous@cvs.gift.sf.net:/cvsroot/gift co giFT

This will download everything you need...

1.1 Updating

To update, enter the directory where all the giFT sources are (/usr/src/giFT or ~/cvs/giFT?) and run:

  $ cvs update -d

All changed files will be updated. It's important you keep your version up to date, since giFT is still in development, and things tend to change rapidly.

2. Compiling and installing

To install giFT/OpenFT enter the newly created giFT directory and do:

  $ ./autogen.sh

This will generate and execute a 'configure' file... if everything goes well (if not, see 2.1), you can do:

  $ su -c 'make install' (if you're not root, or)
  $ make install

... to compile and install giFT, the OpenFT library and the GTK GUI giFT-fe

2.1 Installing without GTK

Not everybody wants to install the GTK client, for example if you want to run giFT on a server. If you do have the GTK libs, you can use './autogen.sh --disable-gtk-client'. But if you don't got those libs you need to copy the file 'm4/gtk.m4' to 'acinclude.m4' first.

2.2 Problems

Lots of people seem to have problems running ./autogen.sh. You need a few programs (and certain versions!) installed to run it successfully. These programs are:

  • automake 1.4+
  • libtool 1.4+
  • autoconf 2.13 (2.52 is buggy!)

If things still don't work (for example you get 'AM' errors) you could also try to download a recent (mostly most recent) version of giFT at http://wideview.33lc0.net/gift/. Grab the latest giFT-YYMMDD.tgz (year, month, day) file. I included my 'Makefile's (generated on a Debian 3.0 system). The(/my) defaults will probably work for most users.

3. Running and using it

Running giFT is easy:

    $ giFT

That'll start the daemon and load all protocols (currently only OpenFT). The daemon will report all kind of status things, but you'll probably only need them when giFT crashes :)

You can also check out some info about the nodes you're connected to and files you share with your browser at localhost:1216, or another port you configured to be your nodepage.

3.1. GUI's

To search for and download files you'll (probably) need a graphical user interface (GUI). At the moment, there are a few available. If you want to write your own, please check the Interface Protocol Documentation.

3.1.1 giFT-fe
The official GUI. It's GTK based and you already have it installed if you installed giFT. Run giFT-setup to configure it.
3.1.2 giFT-shell
Malverians Perl client, previously called gift-perl or 'TooT' (for some mysterious reason :). Its interface is gnut like, and it's an excellent client for all command line junkies. This client is installed by default too. .
3.1.3 giFTcurs
A ncurses interface to giFT, written by weinholt and chnix. More info available at giftcurs.sf.net. You can download it with cvs too (cvs -z3 -d:pserver:anonymous@cvs.giftcurs.sf.net:/cvsroot/giftcurs co giFTcurs)

4. Configuring

This used to be a large part of this guide, but then giFT-setup came, saw and won. It handles the configuration for giFT, OpenFT, giFT-fe and you can/must even configure your 'share paths' with it.

4.1 giFT-setup

Before you run giFT you should configure it. You can do this by using our nifty 'giFT-setup' script. The program is self explanatory, so just run it after installing giFT:

  $ giFT-setup

4.2 nodes file

Ah yeah, the ever popular 'nodes' file... Since a lot of questions in our IRC channel were asked because of this file, it gets a special section here.

Information about OpenFT nodes will be saved in the file ~/.giFT/OpenFT/nodes. This information is vital if you want to connect to the network, so for now a default node is hardcoded in the giFT source.

If you have problems connection to OpenFT, please try to remove the nodes file and restart giFT. A new file (with the default node(s)) will be made. When the default node is down, there probably is a reason for that, so try again later.

But if you can't stand it, and just want to connect, you can always drop in our IRC channel and ask (politely!) for a nodes file.

5. Miscellaneous

A complete new section for all things I don't know where to fit :-)

5.1 Stats

Since OpenFT works with HTTP, it also generates an Apache style 'access.log'. With some tweaking you can make beautiful overviews and graphs with your favorite log analyser / statistics generator. You can find the file in your .giFT directory: ~/.giFT/access.log.
