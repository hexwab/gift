  1. Generic

    • What is giFT?
      giFT is a plugin system that can be used to connect to many different networks. But for now OpenFT is the only network you can use.

    • Where is your IRC channel?
      irc.openprojects.net, join channel #gift

    • History
      Development to create a Linux client to KaZaA was our initial goal. After a lot of reverse engineering and packet sniffing, a group of talented developers which became known as "the givers" stumbled onto our project (at the time "kazaatux"). Within a few weeks, we had a working daemon that could connect to and search the KaZaA network. Shortly after, KaZaA released a new version of their client which eventually led to the breaking of what we now call giFT. After the permanent departure of the givers, the giFT team started moving development into a new direction; foremost was the desire to have a completely open, completely free peer to peer network modeled in the image of FastTrack. Thus OpenFT was born. giFT was moved to a new plugin architecture which would allow the creation of plugins that would make it easy for one client to be compatible with any number of networks. The first of these plugins were to be OpenFT, and a reimplementation of a working FastTrack client.

    • Why use a daemon?
      It creates an easy way for people to focus their time on frontends instead of all of so much networking code. Also, if the daemon changes, all of the frontends will still function the same as they had before. This also makes it so we don't end up with 20 incompatible protocols flying around as standalone clients.

  2. Contribute

    • How can I help speed up development?
      You can't. Well... maybe you can help testing, or make your box available as a search node. Just drop into our IRC channel and ask if you can do something.

    • Front-end Developers
      See the Interface Protocol Documentation

    • Will giFT be ported to any other hosts?
      When the system is stable on Linux/x86, porting to other structures will most likely ensue soon after. But if you would like to experiment with the current code, you're more than welcome. Please report it to our IRC channel if you are successful.

  3. KaZaa, Morpheus, Grokster, oh my!

    • What is this "FastTrack"?
      FastTrack is the company that licenses the library that KaZaA/Morpheus/Grokster operates off of.

    • Why can't I connect to KaZaA/Morpheus/Grokster anymore?
      The encryption has changed, thanks to FastTrack. We have been unable to reverse engineer it as of yet.

    • But I could connect to FastTrack with giFT 0.9.7!
      When version 1.3.3 of KaZaA was released, it contained a "Security Update" which broke our implementation.

    • Are you working on getting back into FastTrack's network?
      Not actively, no. Although quite a few people made attempts to reverse engineer the new encryption, nobody has succeeded, and AFAIK nobody is working on it atm.

    • What about KaZaA's Linux client?
      After KaZaA moved it became unavailable and the old versions can't connect anymore... whatever, it sucked anyway :)

  4. OpenFT

    • What's this OpenFT I've been hearing about?
      It's an Open Source ``clone'' of FastTrack currenty being worked on by the giFT project. We welcome anyone with decentralized/p2p experience to drop by and poke around at our design.

    • Oh, cool, where can I check OpenFT out?
      You can 'check it out' of CVS with these commands:

      cvs -z3 -d:pserver:anonymous@cvs.gift.sf.net:/cvsroot/gift co giFT
      cd giFT
      ./autogen.sh --enable-maintainer-mode
      make
      su -c 'make install'
      giFT

    • Erm... I would like some more info on installing giFT/OpenFT
      Read the Installation guide

    • It just sits there, how the heck do I search for stuff?
      You need a GUI, doh! :) There are a few GUI's available ATM, Installation guide lists them all. Note: The thing that "just sits there" must be running at all times.

    • How come I can't find stuff?
      The network (OpenFT) is still being developed. But, with your help and word of mouth, when the network is in a usable state, we'll take over where KaZaA left off, and eventually bury them in the dirt. Make sure you share _your_ stuff!

    • Is OpenFT eventually going to use features like metadata?
      Yes, eventually. It needs to stabalize first before all sorts of fancy features will be added.

  5. ShadowFT

    • I heard something about something called ShadowFT, what is it?
      It's a program that lets you 'search' for files on the FT network.

    • Cool, where can I get it?
      http://www.dddi.nl/~costar/shadowFT

    • It's neat, how is it doing that? I thought the encryption was changed. How come you guys can't do this?
      It doesn't really connect to the supernodes or central server. You may notice that the real KaZaA client doesn't sit there and scan for IPs for a few hours before you can search. What this program does is scan a ton of IPs for an open port 1214 and uses the KaZaA built in web server (http://server:1214 in browser) to get a list of currently available files, and puts them into a database that it lets you search through locally. Which can explain why alot of times your downloads will fail to even start. Check out the README file with ShadowFT to figure out how to maximize the programs uses.
